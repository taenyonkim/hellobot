- 브랜드 상품코드는 어떻게 정하고 관리하나요?

  - 브랜드에서 관리하는 상품코드라고 되어있는데, 저희가 정하고 전달드리는 방식인가요?
    ▶ 쿠폰번호로 고객이 사용 시도시, 해당 상품이 어떤 상품인지 내부적으로 구분 가능하신 값을 정해주시고, 당사 전문규격에 맞추어 값을 당사 판매상품 셋팅 전에 수기로 전달주시면 됩니다.
    ex. '26년 종합운세' 상품은 브랜드상품코드 10001 / '26년 연애운' 상품은 브랜드상품코드 10002

- 사용자가 입력할 카카오 상품권의 코드 문자열은 특정한 규칙이 있나요?
  ▶ 난수번호이나, 쿠폰번호 앞 두자리 프리픽스 및 쿠폰 자릿수는 지정 가능합니다. 현재 카카오에서 운영 중인 쿠폰번호와 겹치지 않아야 하여 추후 당사에서 카카오 측 발번현황 확인 후 전달드릴 예정입니다.

- 신규 상품권을 등록할 때 테스트는 어떻게 진행하시나요? (실사용 후 취소, 테스트 모드?같은 방식 등)  
   ▶ 당사에서 테스트쿠폰 발급 후, 승인 / 승인취소 테스트 진행 요청드리면 귀사에서 처리 후 당사에서 정상 적용여부 조회하여 확인하게 됩니다.

- 상품권 사용 취소는 어떤 경우인가요? 단순 서비스 내부 오류는 상품을 복구해 주는 형태로도 가능해 보이는데, 사용자가 특별히 상품권의 사용을 복원해달라고 요청하는 경우인가요?
  ▶ 말씀 주신 것과 인증 처리 구간에서 (귀사 내부) 오류 발생시 승인취소 처리가 필요할 수 있으나, 콘텐츠 제공형 상품이다보니 사용 후 취소 불가 정책으로 기본 운영 필요합니다.
  당사에서 정상 사용되지 않은 고객 클레임 인입시, 쿠폰번호 전달드리면 해당 쿠폰의 정상 승인여부 확인하여 전달주시면 됩니다. (등록 계정명 등 고객정보 확인시 일부 마스킹하여 공유요청)

## 1. 기능 정의 (1순위 정보)

### 1-1. 기능 한 줄 요약

> 사용자가 무엇을 할 수 있게 되는지를 1줄로.

```
사용자는 카카오 선물하기에서 받은 쿠폰을 헬로우봇 서비스에 등록하여 재화로 사용할 수 있다.
```

---

### 1-2. 기능의 목적 / 해결하려는 문제

> 비즈니스 관점 — 왜 이 기능을 만드는가.

- 해결하려는 문제: **카카오 선물하기는 국내 거대한 마켓플레이스이고, 노출 영향력이 크다. 우리가 진입하지 못했던 큰 플랫폼에 우리 상품을 노출함으로써 지속적인 신규 사용자 유입이 기대된다.**
- 기대 효과 (정성): 신규 사용자, 기존 사용자가 카카오 선물하기 채널을 통해 쿠폰 구매로 꾸준히 유입될 것이다.
- 가설 (있으면): 카카오 선물하기에 상품이 노출되면 지속적인 신규 사용자 유입이 발생할 것이다.

### 1-3. 기능 카테고리 (`feature-performance-measurement.md` Step 0 4 카테고리)

> 가장 가까운 1개 선택 (복합 기능은 주(主) 카테고리 1개 + 부(副) 카테고리 1~2개).

- [ ] **A. Purchase / Conversion** — 결제·전환

  - [ ] A-3. 구독 결제 (`pay_for_chatbot_subscription`)
  - [V] A-4. **외부 결제 채널** (예: 카카오 쿠폰, 외부 프로모션)
- [V] **B. Content Consumption** — 콘텐츠·스킬 소비

## 7. 메모 / 자유 기술

> 분석 설계 시 알아둬야 할 맥락·제약·우려사항.

```
- 카카오 하트 충전권 쿠폰은 하트 재화로 교환되어 이후 사용자가 하트를 소비하는 시점에 매출로 집계됨 (pay_for_* 이벤트로 집계)
- 카카오 상품 교환권 쿠폰은 100% 할인형 쿠폰으로 교환되어 사용자는 최종 가격 0 상품으로 소비를 진행하게 됨. (구매로 잡히지 않을 것 같음) - 이 부분 이슈 사항
  - 카카오 선물하기로 선물 받아 들어온 사용자의 쿠폰 사용을 집계시 어떻게 처리해야 하나?
    - 상품 교환권 쿠폰 사용시 해당 이용 건은 pay_for_ 로 잡혀야 함 (거래액 집계에 포함되어야 함)
```

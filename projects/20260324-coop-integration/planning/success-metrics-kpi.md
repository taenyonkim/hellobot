# 쿠프마케팅 연동 — Success Metric / KPI 정의서 (초안)

> 작성일: 2026-04-22
> 작성자: /analyze
> 상태: 초안 — 기획·데이터 검토 후 확정

## 0. 목적

카카오 선물하기 상품권 연동의 성과를 객관적으로 측정하고, 운영 의사결정(상품 라인업 조정, 프로모션 집행, 채널 확대)에 활용할 수 있는 KPI 체계를 정의한다.

1pager Success Metric:
- **Input**: 상품권 구매 전환율
- **Output**: 앱 신규 구매자 수, 웹 신규 구매자 수

본 문서는 위 지표를 측정 가능한 형태로 구체화하고, 보조 지표 및 운영 지표를 함께 정의한다.

---

## 1. 지표 체계

### 1.1 North Star (최상위)

| 지표 | 정의 | 산출 방식 |
|------|------|----------|
| **카카오 채널 신규 구매자 수 (MAU)** | 상품권 등록을 통해 첫 헬로우봇 유료 이용이 발생한 사용자 수 (월 단위) | DISTINCT userSeq WHERE 상품권 등록 완료 AND 이전 유료 구매 이력 없음 |

### 1.2 Input Metric (상품권 구매 전환율)

| 지표 | 정의 | 수식 | 집계 단위 |
|------|------|------|----------|
| **상품권 구매→등록 전환율** | 카카오 선물하기에서 판매된 상품권 중 헬로우봇에 등록되어 실사용된 비율 | `(기간 내 등록 완료 건수) / (기간 내 쿠프 발급 건수)` | 일/주/월 |
| **등록 리드타임** | 상품권 발급일~등록일 사이 경과 시간 | 등록 시각 − 쿠프 발급 시각 중앙값/평균 | 월 |
| **등록 성공률** | 상품권 코드 입력 시도 대비 성공 비율 | `성공 건수 / (성공 + 실패 건수)` | 일 |

> ⚠️ 분모(쿠프 발급 건수) 수집 방식 확정 필요 — 쿠프마케팅에서 일별/월별 CSV 또는 API로 수신

### 1.3 Output Metric (신규 구매자)

| 지표 | 정의 | 수식 | 집계 단위 |
|------|------|------|----------|
| **앱 신규 구매자 수** | 상품권 등록 경로로 헬로우봇 첫 유료 이용 경험을 한 앱 사용자 수 | DISTINCT userSeq WHERE 등록 플랫폼 ∈ {iOS, Android} AND 이전 유료 이력 없음 | 월 |
| **웹 신규 구매자 수** | 동일 조건, 플랫폼 = web | DISTINCT userSeq WHERE 등록 플랫폼 = web AND 이전 유료 이력 없음 | 월 |
| **재등록자 수 (보조)** | 2회차 이상 등록 사용자 수 — LTV 추정용 | COUNT userSeq WHERE 등록 건수 ≥ 2 | 월 |

### 1.4 상품 성과 지표 (보조)

| 지표 | 정의 | 집계 단위 |
|------|------|----------|
| **상품별 등록 건수** | product_code별 등록 성공 건수 | 일/월 |
| **상품별 GMV** | product_code별 판매 금액 합계 (= 쿠프 발급 금액) | 월 |
| **상품 타입 믹스** | heart / skill 등록 건수 비율 | 월 |
| **스킬 이용권 소진율** | 스킬 교환권 등록 후 실제 스킬 구매 완료 비율 | 월 |

### 1.5 운영 지표 (대시보드용)

| 지표 | 정의 | 집계 단위 |
|------|------|----------|
| **일별 등록 트렌드** | 등록 건수 시계열 | 시간/일 |
| **에러 코드별 실패 건수** | CM_001~CM_010 발생 빈도 | 일 |
| **구버전 앱 가드 발동 건수** | CO_APP_UPDATE_REQUIRED 응답 수 (ISS-009 관측) | 일 |
| **동시성 경합 발생 건수** | Redlock 대기/실패 횟수 | 일 |
| **L1 사용 승인 평균 응답시간** | 쿠프마케팅 API 레이턴시 | 일 |

### 1.6 정산 지표

| 지표 | 정의 | 집계 단위 |
|------|------|----------|
| **정산 대상 금액** | 월 판매 총액 × 92% (수수료 8% 차감) | 월 |
| **정산 건수 대사** | 쿠프마케팅 L1 사용 내역 vs 내부 `coupc_marketing_coupon_usage` 일치 건수 | 월 |
| **대사 불일치 건수** | 한쪽에만 존재하는 건수 (이상 감지) | 월 |

## 2. 핵심 정의 (합의 필요)

### 2.1 "신규 구매자" 정의

**옵션 A (엄격)**: 헬로우봇 가입 이후 **모든 유료 채널**(IAP, 웹 결제, 기존 쿠폰 등)에서 구매 이력이 없는 사용자가 상품권 등록 시 신규 구매자로 인정
- 기준 테이블: `heart_log` (유료 충전 타입), `payment` 등
- 장점: 진정한 신규 구매 전환 측정
- 단점: 기존 유료 사용자의 상품권 선물 수령도 전환으로 잡아야 하는지 논의 필요

**옵션 B (완화)**: 상품권 등록 후 **1개월 내 신규 가입자**로 한정 (신규 유입 관점)
- 기준: userSeq의 `created_at`이 등록 시각 −30일 이내
- 장점: 채널이 신규 유입을 만들어냈는지 직접 측정
- 단점: 기존 헬로우봇 사용자가 받은 상품권은 제외됨 → 상품권의 재구매 효과 누락

**옵션 C (복합, 권장)**: 두 지표를 병행
- **"신규 가입 구매자"** (옵션 B) + **"첫 유료 경험 사용자"** (옵션 A) 를 별도 컬럼으로 집계
- 대시보드에서 동시 노출

👉 **기획 팀 확정 필요 항목**

### 2.2 전환율 분모 (쿠프 발급 데이터)

**필요 데이터**:
- 발급일, ProductCode, 발급 수량, 판매액
- 가급적 쿠폰번호 단위(L0 조회 가능한 식별자) — 1:1 대사를 위해

**수령 방식 후보**:
| 방식 | 주기 | 난이도 | 비고 |
|------|------|--------|------|
| 일별 CSV (이메일/SFTP) | D+1 | 낮음 | 가장 현실적, 수동 업로드 또는 SFTP→GCS |
| 쿠프마케팅 발급 조회 API | 실시간 | 중간 | API 스펙 확인 필요. 현재 연동 API 문서에는 사용(L1) 관련만 존재 |
| 월별 정산 파일 | M+10 | 낮음 | 정산용으로만 가능. 일간 모니터링 불가 |

👉 **쿠프마케팅 측과 협의 필요 항목**

### 2.3 어트리뷰션 Lookback

- 상품권 **등록 시점**을 성과 귀속 시점으로 사용 (발급 시점 아님)
- 이유: 발급~등록 사이에 선물 전달/보관 지연이 있을 수 있음
- Lookback 기간: 등록일 기준 **30일** 내 첫 구매를 "전환" 으로 볼지, 등록 행위 자체를 전환으로 볼지 → **등록 = 전환**으로 단순화 권장 (상품권 자체가 구매 행위이므로)

## 4. 대시보드 구성안

### 4.1 경영진 요약 (월 단위)
- 신규 구매자 수 (앱/웹) — 월 트렌드
- 전환율 (발급 대비 등록)
- 정산 대상 금액
- 상품별 판매 Top 5

### 4.2 운영 대시보드 (일 단위)
- 일별 등록 건수 (상품별, 플랫폼별 스택)
- 에러 코드 분포
- 구버전 가드 발동 건수 (ISS-009)
- L1 API 레이턴시 p50/p95

### 4.3 상품 성과 (월 단위)
- 상품별 등록 건수/GMV
- 스킬 이용권 → 스킬 구매 소진율
- 재등록 사용자 분포

👉 **대시보드 도구 결정 필요** (Metabase / Looker / Spreadsheet + BQ)

## 1. 네이밍 규칙 (제안)

### 1.1 형식

```
K{TYPE}{SEQ:5}
│  │     └── 5자리 일련번호 (00001부터 시작)
│  └──────── 상품 타입 (H=Heart, S=Skill)
└─────────── 헬로우봇 고정 프리픽스 (K = Kakao 상품군)
```

**예시**:

- `KH00001` — Kakao Heart 1번 상품 (하트 25)
- `KS00001` — Kakao Skill 1번 상품 (스킬 교환권)

### 1.2 규칙 상세

| 항목     | 규칙                      | 비고                                            |
| -------- | ------------------------- | ----------------------------------------------- |
| 총 길이  | 7자리 고정                | VARCHAR(50) 범위 내                             |
| 프리픽스 | `K` 고정                  | 향후 타 채널 확장 시 `N`(네이버) 등 구분 용이   |
| 타입     | `H` (Heart) / `S` (Skill) | 향후 `B`(번들) 등 확장 가능                     |
| 일련번호 | 001~99999 제로패딩        | 단종·재출시 시 **신규 번호 발급** (재사용 금지) |
| 대소문자 | 대문자 고정               |                                                 |

### 1.3 단종/변경 정책

- 상품 단종: `is_active = false`로 비활성화. ProductCode는 **재사용하지 않음**.
- 가격/내용 변경: 기존 ProductCode 비활성화 + 신규 ProductCode 발급 (사용 이력 추적성 유지)
- 명칭만 변경: 기존 ProductCode 유지 가능

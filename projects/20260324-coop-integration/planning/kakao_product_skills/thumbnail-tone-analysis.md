## 2. 평가 프레임 — 카카오 선물하기 매대용 6축

[skill-priority-analysis.md §1](./skill-priority-analysis.md) 의 4축은 SKU 선정용. 섬네일 평가는 **그리드 노출 → 클릭 → 구매 → 발송** 동선용 별도 6축으로 정의.

| # | 평가축 | 정의 | 가중치 |
|---|--------|------|--------|
| T1 | **선물 카드 시각 언어 정합** | 카카오 선물하기 매대에서 옆에 진열될 기프티콘·웹툰·이모티콘 교환권과 톤이 맞는가 | ★★★ |
| T2 | **그리드 가독성** | 1/3 화면 사이즈로 작게 노출됐을 때 핵심 메시지가 잡히는가 | ★★★ |
| T3 | **받는 사람 수용 부담** | 캡쳐해서 친구한테 보낼 만큼 부담 없이 받을 수 있는가 | ★★★ |
| T4 | **콘텐츠 본질 전달** | 어떤 운세/콘텐츠인지 즉시 이해되는가 | ★★ |
| T5 | **헬로우봇 브랜드 식별성** | 헬로우봇 상품임이 분명한가 | ★ |
| T6 | **구매 의사결정 정보** | 가격·구성 등 결정에 필요한 정보가 있는가 | ★ |

> T3 가중치 ★★★ 인 이유는 [SKU 선정 프레임의 B 축](./skill-priority-analysis.md#1-분석-프레임워크)과 동일. 카카오 선물하기는 노출형 채널 — 받는 사람이 부담 느끼면 미사용·환불.

## 4. 정성 코멘트

### 4.1 안 A — 결정적 약점은 "수신자 시점" 부재

- 안 A 는 **헬로우봇 앱에 직접 들어와 자기 운세 보러 온 사용자**에게는 정확한 톤. "상위 4% 이상형", "안정형 데이트뷰" 같은 결과 티저가 클릭률을 끌어올림.
- 그러나 카카오 선물하기는 **수신자의 시점에서 자신을 노출하는 카드**. 받자마자 카톡 친구창에 미리보기로 깔리고, 받는 사람이 캡쳐해 다른 친구에게 보내기도 함.
- "운빨충", "이상형 스포" 같은 어휘는 받는 사람이 보기에 "내가 이런 사람이라고 친구가 보낸 거야?" 라는 의구심을 만듦. 본인이 직접 산 거면 자조 개그가 되지만, **받은 거면 라벨링**.
- "데이트뷰", "안정형" 같은 내부 시스템 용어가 외부 매대에 노출되는 것도 문제. 사용자는 이 용어를 모름.

### 4.2 안 B — 약점은 "차별화"와 "콘텐츠 호기심"

- 안 B 는 카카오 매대 시각 언어에 완전히 동조하면서 수신자 부담을 제거 — KS00028 같은 T3 SKU 를 굳이 진열한다면 이 톤이어야 함.
- 다만 **하트 + 인물 사진 + 보라 그라데이션** 조합은 카카오 매대에서 흔한 비주얼이라 그리드에서 옆 상품들과 묻힐 위험. 헬로우봇 고유성을 살릴 차별화 포인트(점성·사주 모티프, 한국 전통 패턴, 헬로우봇 캐릭터 등)가 빠져 있음.
- 인물 사진의 의도가 약함 — 결혼·이직 같은 C1 컨텍스트에는 인물 사진이 부담스러울 수 있고, 재미 SKU 에서는 "이 인물이 내 운명?" 의도로 살아남. 컨텍스트별로 인물 사진 활용 여부 분리 필요.

### 4.3 카카오 선물하기 매대 가설 시뮬레이션

```
[그리드에서 옆 상품들과 나란히 노출될 때]

스타벅스 아메리카노 ₩4,500   배스킨라빈스 파인트 ₩9,900
[기프티콘 정형]              [기프티콘 정형]

배달의민족 1만원권 ₩10,000   요기요 5천원권 ₩5,000
[기프티콘 정형]              [기프티콘 정형]

      → 옆에 안 A 가 놓이면: "어? 이건 운세 결과 같은데?" (왜 매대에 있는지 의문)
      → 옆에 안 B 가 놓이면: "디지털 교환권이구나" (즉시 카테고리 인지)
```

## 5. 권장 결정 — 안 B 톤을 베이스로

### 5.1 결정 사항

- 카카오 선물하기 매대용 섬네일 **베이스 톤 = 안 B**.
- 단, 안 B 톤을 그대로 모든 SKU 에 복제하지 말 것. **컨텍스트별 변형이 필요** ([§6](#6-컨텍스트별-변형-가이드) 참조).
- 안 A 톤은 **헬로우봇 앱 인앱 노출용**으로 유지 — 둘 다 자산이지만 사용처가 다름.

### 5.2 안 B 가 보강해야 할 사항

| 항목 | 보강 방향 | 우선순위 |
|-----|---------|----------|
| **헬로우봇 차별화** | 보라·하트 외에 헬로우봇 고유 모티프(사주 격자, 음양오행 패턴, 캐릭터 등) 1개 도입 | 높음 |
| **타이틀 가독성** | "연애 상대 스포 / : 내 사주 존잘 존예" 구분 기호 ":" 위치 어색 → 한 줄로 정리 ("내 사주 존잘 존예 미리보기" 등) | 중간 |
| **인물 사진 의도** | 인물 사진을 쓰는 SKU/안 쓰는 SKU 가이드 필요. C1 축하 라인은 인물 대신 모티프(반지·집·축하 화환 등) 권장 | 높음 |
| **가격 표기 통일** | "18,000원" → "₩18,000" 또는 매대 표준 표기 확인 후 통일 | 낮음 |

## 6. 컨텍스트별 변형 가이드

안 B 의 골격(카드 비주얼 + 가격 표시 + 헬로우봇 로고)을 유지하면서 **컨텍스트별로 색감·모티프·인물 활용을 분기**한다. [skill-priority-analysis.md §2 5분류](./skill-priority-analysis.md#2-선물-컨텍스트-5분류) 대응.

| 컨텍스트 | 베이스 컬러 | 모티프 | 인물 사진 | 톤 키워드 |
|---------|-----------|-------|----------|----------|
| **C1 축하/축복** (결혼·이직·이사·합격) | 골드 + 네이비 + 화이트 | 반지·집·축하 화환·트로피·꽃잎 | ✗ (없음) | 단정, 정중, 고급감 |
| **C2 응원** (직장·짝사랑) | 코랄 + 옐로 + 라이트 그린 | 응원봉·하트·새싹·해 | △ (선택) | 따뜻, 가벼움 |
| **C3 신년 시즈널** | 레드 + 골드 | 띠 동물(말)·복주머니·매화 | ✗ | 시즌감, 한국적 |
| **C4 재미·깔아두기** | **보라 + 그라데이션 (현재 안 B)** | 하트·별·이모지 | ◎ (적극 활용) | 발랄, 디지털, MZ |
| **C5 셀프 (임신 준비 등)** | 파스텔(베이비 핑크·민트) | 아기 발자국·구름 | ✗ | 부드러움, 셀프 동기 |

KS00028 은 **C4 재미** 컨텍스트 — 현재 안 B 톤이 정확히 맞는다.

> **주의**: 사전 분석에서 KS00028 은 Tier 3(직접 선물 부적합 가까움)로 분류됨. 1차 출시 SKU 가 아니라면 이 섬네일은 디자인 톤 검증용 샘플인지, 1차 출시 후보로 승격된 것인지 사용자 확인 필요. 만약 1차 10개 중 하나라면 [skill-priority-analysis.md §4](./skill-priority-analysis.md#4-tier-1-1차-출시-10개-sku-선정-근거)의 컨텍스트 커버리지 재조정이 필요.

## 7. A/B 테스트 권장 (선택)

리소스가 허락한다면, 1차 출시 시점에 **KS00028 한 SKU 로 안 A·B 둘 다 매대에 노출** 후 클릭률·전환율 비교가 가장 빠른 검증.

| 가설 | 검증 지표 | 비교 기간 |
|-----|---------|----------|
| H1: 안 B 가 안 A 보다 노출→클릭 전환율 1.5배 이상 | 카카오 매대 CTR | 2주 |
| H2: 안 B 가 안 A 보다 미사용률(쿠폰 등록 안 함) 낮음 | 발송→쿠폰 등록 전환율 | 1개월 |
| H3: 안 B 가 안 A 보다 캡쳐·공유 빈도 높음 (정성) | 인스타·X 검색 시 캡쳐 후기 수 | 1개월 |

다만 카카오 선물하기 매대가 한 SKU 에 섬네일 2개를 동시 노출하는 걸 허용하는지 확인 필요. 불가하면 출시 2주 후 섬네일 교체 → 전후 비교(노이즈 크지만 가능).

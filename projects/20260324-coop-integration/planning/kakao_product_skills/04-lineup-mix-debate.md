### 1.2 SKU 선정 기준 (data/skills-by-segment-12m.csv 활용)

기존 헬봇 SKU 중 **카카오에 적합한 14개**를 선정. 선정 기준:

| 기준 | 측정 | 데이터 컬럼 |
|---|---|---|
| **A. 매출 검증** | 12M 매출 ≥ 임계값 | `revenue_12m` |
| **B. 세그먼트 정합** | C1(26-35) 결제자 비중 ≥ 50% (메인) / C3(16-25) 비중 ≥ 40% (라이트) | `age_26_30` + `age_31_35` / `age_16_20` + `age_21_25` |
| **C. 가격대 정합** | ₩9,900 / ₩17,000~22,500 / ₩29,900~37,500 3개 가격 라인 | `price_amount` |
| **D. 콘텐츠 보편성** | topic ∈ {연애·총운·결혼·일반운세·가족자녀·학업직업·재물금전} | `topic` |
| **E. 메시지·섬네일 가공 가능성** | 카피 변환 / 섬네일 재제작 가능한 SKU | (수기 판단) |
| **F. 자사몰 자기잠식 위험** | 12M 매출 매우 높은 SKU는 채널 차별화 강화 필요 (R6 대응) | `revenue_12m` 상위 |

→ 14 SKU 선정은 별도 작업 (data/skills-by-segment-12m.csv 정렬 + 수기 검증). `/dev-data` 또는 PM이 직접 후보 압축.

### 1.3 콘텐츠 보편성 (03번 메모 결론 유지)

## 3. 카피·섬네일 가공 가이드라인

`[결정 0.2·0.3 실행 가이드]`

### 3.1 카카오 노출 상품명 (카피 변환) 원칙

| 원칙 | 인앱 톤 | 카카오 톤 | 비고 |
|---|---|---|---|
| 페르소나형 → 메시지형 | "운빨충 친구에게" | "친구의 한 해 운세, 함께 살펴봐요" | M2 즉시 이해도 [v2 §3.3](./skill-priority-analysis-v2-market-fit.md) |
| 자조·개그 → 진심·축하 | "사직서 품은 너에게" | "이직을 준비하는 친구를 위해" | 받는 사람 라벨링 회피 |
| 결과 스포 → 콘텐츠 예고 | "상위 4% 존잘 사주" | "내 사주가 알려주는 연애 매력" | 자극 ↔ 안전성 균형 |
| 단순 콘텐츠 → 선물 맥락 | "2026년 신년운세 보고서" | "친구의 새해, 1년의 길을 미리" | 선물 동기 트리거 |

### 3.2 카피 변환 예시 (5개 SKU)

| SKU 카테고리 | 인앱 상품명 (예) | 카카오 변환 (예) |
|---|---|---|
| 신년운세 | "2026년 신년운세 보고서" | "친구의 새해, 1년 12달의 운을 선물하세요" |
| 결혼 궁합 | "결혼 OX 궁합: 이 사람과 결혼해도 될까?" | "결혼을 축하해 — 두 사람의 사주가 만나는 길" |
| 이직·직업 | "사주 직업 컨설팅: 사업/직장/업종" | "이직을 준비하는 친구를 위한 사주 컨설팅" |
| 학부모 자녀 | "[학부모 필독] 내 아이의 재능과 운명" | "아이의 빛나는 재능, 1년의 길을 함께" |
| 라이트 연애 | "걔는 나를 연애 상대로 볼까?" | "친구의 썸, 가벼운 한 마디" |

→ 카피 가공 규칙 SSOT는 [`skill-priority-analysis-v2-market-fit.md §3.3 M2`](./skill-priority-analysis-v2-market-fit.md) 와 정합 (메시지형 통일).

### 3.3 섬네일 카카오 플랫폼 가이드라인

| 가공 영역 | 인앱 표준 | 카카오 표준 |
|---|---|---|
| 가격 노출 | 카드에 가격 표시 | **가격 노출 제거** (매대 별도 표시) |

| 시각 언어 | 인앱 자체 톤 | **카카오 디지털 교환권 시각 언어 정합** ([thumbnail-tone-analysis](./thumbnail-tone-analysis.md) §1 B 안) |

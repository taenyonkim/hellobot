## 3. SKU 자체 평가 (K1) — A/B 무관, 콘텐츠 성격

K1 은 섬네일 디자인이 아닌 SKU 의 본질을 보는 축.

### 3.1 운세 콘텐츠 시장에서의 위치

| 카테고리 | 시장 규모 추정 | 이 SKU 위치 |
|---------|--------------|------------|
| 연애운·궁합 | ★★★★★ (압도적) | 본 SKU 비해당 |
| 재물운 | ★★★★ | 비해당 |
| 신년운세 | ★★★★ (시즌) | 비해당 |
| 사주 풀이 (총운·일반) | ★★★ | 비해당 |
| **지역·풍수·이사** | **★★ (니치, 그러나 트리거 명확)** | **본 SKU 정확히 매칭** |
| 자녀운 / 임신운 | ★★ | 비해당 |

**결론**: 운세 콘텐츠 시장 절대 규모로는 **중하위 니치 카테고리**. 하지만:
- 카카오 선물하기에서 "이사·집들이 선물" 매대에 노출되면 **운세 카테고리 내 경쟁 없음**. 다른 운세 SKU 가 들어오지 못하는 시장.
- 트리거 이벤트(이사·자취·이직·청약·신혼집)가 명확해 "지금 이걸 줘야 한다"는 동기가 강함. 연애운·재물운보다 충동적 깔깔이가 아닌 **목적 있는 선물**.
- 1차 출시 10개 안에서 **C1(인생 이벤트 축하) 라인의 핵심 SKU 역할**.

> **[A.7 내부 실측 정합성 확인 (2026-05-17 추가)]** — [skill-launch-roadmap §0.A.7](../../skill-launch-roadmap.md#a7-참고5-헬봇-앱-내부-세그먼트-eda--우리-사용자-실측-app-only-30일)
> - KS00020 핵심 발신 세그먼트(이사·집들이) = 30대 = 헬봇 결제 코어. A.7.1 **26-30 매출 81M / 31-35 매출 63.8M, 두 구간이 전체 매출 76% 점유**
> - 사주·풍수 콘텐츠 분류 — A.7.2 **31-35 사주 비중 77.2%, 36-40 81.7%** → 본 SKU의 풍수 정통성이 결제 코어 콘텐츠 영역에 직접 포함
> - 단가 ₩17,000 — A.7.1 26-50 ARPPU 23K 평탄대 안의 라이트 가격대. 발신 부담 낮음
> - 결론: **K1 부분 부합 + A.7 내부 실측 정합** → 1차 출시 라인업 포함 정당화 재확인

### 3.2 사용자 기준 1번에 대한 답

> "카카오 선물하기 선물에서 사주 운세 콘텐츠 시장 많은 순"

**부분 부합**. 운세 콘텐츠 절대 시장 규모로는 큰 카테고리가 아니지만, **선물 트리거가 명확한 카테고리에서는 1위급**. 1차 출시 10개에는 반드시 포함되어야 함 (이미 [skill-priority-analysis.md §4 Tier 1](../../skill-priority-analysis.md#4-tier-1-1차-출시-10개-sku-선정-근거)에 포함됨).

> 만약 K1 기준만으로 시장 절대 규모를 우선시한다면 KS00013(결혼 궁합), KS00001(신년운세), KS00011·12(커플 궁합)이 먼저. KS00020은 **C1 라인업의 다양성 확보**용으로 의미가 큼.

## 5. KS00028 케이스와 정반대 결론 — 왜?

[thumbnail-tone-analysis.md](../../thumbnail-tone-analysis.md) 에서는 KS00028 "연애 상대 스포"의 경우 B(선물 카드형)가 우세였다. 같은 디자인 톤이 SKU 별로 정반대 결과를 내는 이유는 SKU 의 **선물 가치 구조가 다르기 때문**.

| 구분 | KS00028 연애 상대 스포 | KS00020 운이 트이는 동네 |
|-----|---------------------|----------------------|
| 컨텍스트 | C4 재미·깔아두기 | **C1 인생 이벤트 축하 (이사·집들이)** |
| 핵심 가치 | "킬킬 재미" — 가벼운 호기심 | **콘텐츠의 구체적 정보 가치** — 이사 결정에 도움 |
| 받는 사람 부담 | 자극적 페르소나로 부담 ↑ → 단순 카드형으로 톤다운 필요 | 정보 가치 노출이 부담을 오히려 해소 ("내가 진짜 받을 만한 선물이네") |
| 결과 노출 | 결과 노출 시 받는 즉시 "라벨링" 위험 ("내 외모가 어떻다고?") | 결과 노출이 콘텐츠 매력 → 호기심 ↑ |
| 매대 옆 상품 | 깔깔이 기프티콘과 묶임 → 카드형이 정합 | 집들이·이사 선물 매대 → 정보형이 정합 |
| **베스트 톤** | **B (선물 카드형)** | **A (정보 집약형)** |

> 핵심 인사이트: **컨텍스트별로 톤이 다른 게 맞다**. [thumbnail-tone-analysis.md §6 컨텍스트별 변형 가이드](../../thumbnail-tone-analysis.md#6-컨텍스트별-변형-가이드) 의 매트릭스가 작동한다는 것이 KS00020 으로 확인됨.

## 6. 섬네일 A 가 보완할 사항

A 가 우세하지만 그대로 출시하기 전에 다듬을 점.

### 6.1 결과 카드 구성 (필수)

현재 A 의 결과 카드 3개 중 1개가 부정 결과:
- "서울 성동구 / **근무지 BEST** / 이직 명예 2배 상승" — 긍정
- "경기도 분당 / **거주지 BEST** / 3년 안에 내 집 마련" — 긍정
- "서울 동대 / **연애 WORST** / 옛날 연인 만나 미련" — **부정**

**문제**: 카카오 선물하기 매대 노출 시점에 받는 사람 시점에서 "내 결과도 WORST 일 수 있겠네"라는 두려움 발생. 콘텐츠를 풀어 본 사람은 BEST/WORST 둘 다 가치 있음을 알지만, 매대에서 처음 보는 사람에게는 부정 결과 노출 = **선물 부담 ↑**.

**권장**:
- 결과 카드 3개 모두 **BEST/긍정 톤**으로 통일
- 또는 부정 카드를 **중립 톤** ("서울 동대 / 인연 GOOD / 새로운 만남 미리보기")으로 재구성
- WORST 결과의 가치(피해야 할 곳 알려주는 기능)는 상세페이지에서만 노출

### 6.2 그리드 가독성 보강

A 는 정보 8개(타이틀·결과카드 3개·해시태그 3개·로고)가 경쟁. 그리드(1/3 화면)에서 작게 노출되면 흐려진다.

**권장**:
- 결과 카드 3개 중 2개만 노출 (BEST 2개)
- 해시태그를 카드 하단 1줄로 축약
- 타이틀 "내 운이 트이는 동네" 폰트 크기 키우기

### 6.3 가격 표기 추가

[thumbnail-tone-analysis.md §3](../../thumbnail-tone-analysis.md#3-6축-평점) 에서 짚었듯이 가격 표기는 결정 비용 ↓.

**권장**: B 의 좌하단 가격 표기 영역을 A 에도 도입 ("17,000원").

### 6.4 통합안 — A·B 의 베스트 조합

```
[A 결과 미리보기 카드 (2개)]
    +
[B 한국지도 + 핀 모티프]
    +
[B 가격 표기 영역]
    +
[B 한국적 구름 모티프 (배경 데코)]
    +
[A 해시태그 (1줄로 축약)]
```

세 번째 변형 시안(C)으로 발주 권장.

## 7. 섬네일 B 의 잔류 가치

B 가 우세는 아니지만 폐기하지 말 것. 다음 용도가 있다.

| 용도 | 이유 |
|-----|------|
| **카카오 매대 페이지네이션 2번째 슬라이드** | 카카오 선물하기는 상품 카드를 2~3장 슬라이드로 노출하는 사례 있음. 1슬라이드 A(정보형) → 2슬라이드 B(가격·카드형)로 보완 |
| **검색/카테고리 노출용 작은 썸네일** | 매우 작게 노출되는 영역(검색 결과 등)에서는 A 가독성 ↓, B 가 효과적 |
| **인스타·페북 광고 소재** | SNS 피드에서는 카드형(B)가 광고 톤에 맞음. 정보형(A)는 광고로 가면 "정보 광고" 느낌 |

## 8. 상세페이지 리뷰

> 이미지 길이가 매우 길어 전체 디테일은 직접 확인 권장. 식별 가능한 구성 기반으로 코멘트.

### 8.1 강점

- **인앱 콘텐츠 결과 카드 + 후기**를 길게 노출 — 헬로우봇 모르는 사용자에게 "이 정도 깊이 있는 콘텐츠"임을 충분히 입증 (K2 보강)
- 모바일 세로 스크롤 형식 — 카카오 선물하기 상세페이지 표준에 맞음

### 8.2 우려

- **길이**: 매우 길어 사용자가 끝까지 스크롤할지 의문. 핵심 정보(결과 미리보기 → 후기 → 콘텐츠 구성)가 **상단 30%** 안에 모두 들어와야 함
- **반복**: 결과 카드/후기가 여러 번 반복되는 인상. 카카오 매대 상세페이지에서는 **반복 최소화 + 정보 밀도 조정** 권장
- **콘텐츠 후반부**: 이미지가 너무 작아 식별 불가. 콘텐츠 구성/이용 안내/생년월일 입력 안내가 들어가는 영역으로 추정 — 사용 방법(쿠폰 등록 → 앱 다운 → 사용)이 명확한지 확인 필요

### 8.3 카카오 선물하기 매대 표준 검증 필요

- 상세페이지 최대 길이/높이 제한 확인
- 가격·유효기간 표기 위치 가이드라인 확인
- 이용 방법 섹션 필수 여부 확인

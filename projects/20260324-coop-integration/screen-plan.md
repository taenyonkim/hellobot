## 2. 진입점 — ✅ 확정

**기존 쿠폰 탭 재사용** + 카카오 딥링크 지원

| 경로 | 설명 |
|------|------|
| 프로필 → 쿠폰 탭 | 기존 쿠폰 입력란을 그대로 사용 |
| 카카오 선물하기 딥링크 | 카카오에서 직접 쿠폰 화면으로 랜딩 |

- 별도 화면 신규 개발 없음
- 서버가 쿠폰 코드 형식을 판별하여 **기존 쿠폰**이면 기존 처리, **카카오 상품권**이면 쿠프마케팅 조회 API 호출

## 3. 화면 상세

### 쿠폰 화면 (기존 화면 재사용)

**목적**: 사용자가 쿠폰 코드를 입력하고 보유 쿠폰 목록을 확인하는 화면

**화면 유형**: 네이티브 (iOS / Android / Web)

**구성 요소**:

| 요소 | 설명 |
|------|------|
| 헤더 | "쿠폰 {N}" (보유 쿠폰 수) |
| 입력란 | 플레이스홀더: "쿠폰 코드를 입력해주세요" |
| 등록 버튼 | 입력 시 활성화 (분홍색), 비어있으면 비활성화 (회색) |
| 힌트 텍스트 | "헬로우봇 쿠폰, 카카오 상품권 등 쿠폰 번호를 입력해보세요." |
| 쿠폰 리스트 | 보유 쿠폰 카드 목록 (기존 + 스킬 이용권) |

**인터랙션**:
- 입력 필드 비어있으면 등록 버튼 비활성화
- 등록 탭 → 로딩 스피너 → 서버 판별 → 상품 유형별 확인 팝업 또는 에러 팝업

---

### S2-A. 하트 충전권 확인 팝업

**목적**: 조회 성공 시, 하트 충전권 상품 정보를 보여주고 사용 여부를 확인받는다

**표시 조건**: 조회 성공 + productType = `heart`

**구성 요소**:

| 요소 | 설명 | 데이터 출처 |
|------|------|------------|
| 하트 이미지 | ♥ 아이콘 (분홍 배경 원형) | 공통 에셋 |
| 상품명 | "하트 충전권 5천원권" | `coupc_marketing_product.product_name` |
| 충전 수량 | "♥ 500개" (강조 표시) | `coupc_marketing_product.heart_quantity` |
| 안내 문구 | "이 쿠폰을 사용하면 하트 N개가 충전됩니다. 사용하시겠습니까?" | — |
| 취소 버튼 | 팝업 닫기, 쿠폰 미사용 | — |
| 사용하기 버튼 | L1 사용 API 호출 → 하트 충전 | — |

**인터랙션**:
- 사용하기 탭 → 버튼 로딩 + 비활성화 (중복 탭 방지) → 성공 시 S3 / 실패 시 S5
- 취소 탭 → 팝업 닫기

---

### S2-B. 스킬 교환권 확인 팝업

**목적**: 조회 성공 시, 교환할 스킬 정보를 보여주고 스킬 이용권으로 교환할지 확인받는다

**표시 조건**: 조회 성공 + productType = `skill`

**구성 요소**:

| 요소 | 설명 | 데이터 출처 |
|------|------|------------|
| 배너 이미지 | 해당 스킬의 배너 이미지 | 스킬(chatbot) 배너 URL |
| 상품명 | "(스킬명) 교환권" | `coupc_marketing_product.product_name` |
| 챗봇명 | "챗봇: (챗봇명)" | chatbot 정보 |
| 안내 문구 | "이 쿠폰을 스킬 이용권으로 교환합니다. 사용하시겠습니까?" | — |
| 취소 버튼 | 팝업 닫기 | — |
| 사용하기 버튼 | L1 사용 API → 100% 할인 쿠폰 발급 | — |

**인터랙션**:
- 사용하기 탭 → 버튼 로딩 + 비활성화 (중복 탭 방지) → 성공 시 S4 / 실패 시 S5
- 취소 탭 → 팝업 닫기

---

### S3. 하트 충전 완료 팝업

**목적**: 하트 충전 완료를 알리고, 내 하트 페이지로 이동하여 확인할 수 있게 한다

**구성 요소**:

| 요소 | 설명 |
|------|------|
| 완료 아이콘 | ✓ (초록 원형 배경) |
| 메시지 | "하트 {N}개가 충전되었어요" |
| 충전 확인하기 버튼 | **내 하트 페이지로 이동** |

**인터랙션**:
- 충전 확인하기 탭 → 쿠폰 화면 닫기 → **내 하트 페이지**로 이동 (충전 내역 확인)

---

### 내 하트 페이지 (기존 화면)

**목적**: 하트 충전 결과를 확인

- 기존 내 하트 페이지 그대로 활용
- 보유 하트 수량이 충전분 반영되어 표시
- 충전 내역 최상단에 **"카카오 상품권 하트 충전"** 항목 노출 (하이라이트)

---

### S4. 스킬 이용권 등록 완료

**목적**: 스킬 이용권(100% 할인 쿠폰) 발급 완료를 알리고, 쿠폰 리스트에서 바로 확인할 수 있게 한다

> **팝업이 아닌 토스트 + 리스트 업데이트 방식**

**처리 순서**:
1. S2-B 팝업 닫힘
2. 쿠폰 입력란 초기화
3. 쿠폰 리스트 **최상단에 스킬 이용권 카드 추가**
4. 상단 **토스트 알림**: "스킬 이용권이 등록되었어요" (2.5초 후 자동 사라짐)
5. 헤더 쿠폰 수 업데이트 ("쿠폰 2" → "쿠폰 3")

**스킬 이용권 카드 구성**:

| 요소 | 설명 |
|------|------|
| 배지 | "스킬 이용권" (보라색) |
| 이름 | "(스킬명) 이용권" |
| 설명 | "챗봇: (챗봇명)" |
| 유효기간 | "YYYY.MM.DD HH:mm 까지" |
| 카드 스타일 | 보라색 테두리, 연보라 배경 (일반 쿠폰과 시각적 구분) |

**인터랙션**:
- 이용권 카드 탭 → **스킬 상세 페이지**로 이동 (쿠폰 적용 상태)

---

### 스킬 상세 페이지 (기존 화면 + 쿠폰 적용)

**목적**: 스킬 이용권이 적용된 상태로 스킬 정보를 확인하고 이용을 시작할 수 있게 한다

- 기존 스킬 상세 페이지 UI 그대로 활용
- **가격 표시 변경**: 원래 가격 취소선 + **♥ 0** 강조 + "쿠폰 적용" 배지
- 하단 CTA: **"지금 보러가기"** (기존 구매 버튼 → 쿠폰 적용으로 무료)

**인터랙션**:
- 지금 보러가기 탭 → **챗봇 채팅방**으로 이동, 쿠폰 자동 적용되어 무료 이용
- 뒤로가기 → 쿠폰 화면으로 복귀

---

### 챗봇 채팅방 (기존 화면)

**목적**: 스킬 이용권 사용으로 스킬이 자동 시작된 채팅방

- 기존 챗봇 채팅방 UI 그대로 활용
- 해당 스킬이 **자동 시작**됨
- "쿠폰 적용 · 무료로 이용 중" 인디케이터 표시 (중앙 배지)
- 별도 하트 차감 없이 진행

---

### S6. 로그인 안내 팝업

**목적**: 미로그인 사용자가 쿠폰 입력을 시도할 때 로그인의 필요성을 안내한다

**표시 조건**: 미로그인(anonymous) 상태 + 입력란 포커스 + 세션 내 미거절

**구성 요소**:

| 요소 | 설명 |
|------|------|
| 경고 아이콘 | ! (빨간 원형 배경, 24px) |
| 제목 | "로그인하지 않고 쿠폰을 등록하면 위험해요" |
| 설명 | "소중한 선물이 사라질 수 있어요 꼭 로그인하고 입력해 주세요" |
| 로그인하기 버튼 | 노란 버튼 (#FFE967), 117px, rounded-26 |
| 다음에 할래요 | 흰 텍스트 링크, 팝업 바깥 하단 |

**인터랙션**:
- 로그인하기 탭 → 로그인 페이지 이동 (fallbackUrl=/coupon)
- 다음에 할래요 탭 → 팝업 닫기, 세션 내 재노출 안 함
- 오버레이(dim) 클릭 → 닫히지 않음 (반드시 버튼으로만 닫기)
- 세션 내 1회만 노출 (거절 시 sessionStorage에 기록)

---

### S5. 에러 팝업

**목적**: 쿠폰 조회 실패 또는 사용 처리 실패 시 원인을 안내한다

**구성 요소**:

| 요소 | 설명 |
|------|------|
| 에러 아이콘 | ! (빨간 원형 배경) |
| 에러 메시지 | 상황별 메시지 (아래 표) |
| 확인 버튼 | 팝업 닫기 |

**에러 유형별 메시지**:

| 에러 코드 | 메시지 |
|----------|--------|
| `8001`, `8002` | 유효하지 않은 쿠폰번호예요. 쿠폰번호를 다시 확인해주세요. |
| `8003` | 기간이 만료된 쿠폰이에요. |
| `8005` | 이미 사용된 쿠폰이에요. |
| `8099` | 결제가 취소된 쿠폰이에요. |
| `9981`~`9999`, 타임아웃 | 일시적인 오류가 발생했어요. 잠시 후 다시 시도해주세요. |
| 내부 (상품 매핑 실패) | 현재 사용할 수 없는 쿠폰이에요. |
| 내부 (L2 롤백 후) | 오류가 발생하여 쿠폰이 원복되었어요. 다시 시도해주세요. |

**인터랙션**:
- 확인 탭 → 팝업 닫기 → 쿠폰 화면 복귀 (입력 필드 유지, 재시도 가능)

## 4. 화면 전환 상세

### 4-1. 정상 플로우 — 하트 충전권

```
쿠폰 화면
  │ 코드 입력 → [등록] 탭
  │ → 로딩 스피너
  │ → 조회 API (서버가 카카오 상품권으로 판별)
  ▼
S2-A 하트 확인 팝업 (오버레이)
  │ [사용하기] 탭
  │ → 버튼 로딩 (중복 탭 방지)
  │ → L1 사용 + 하트 충전 API
  ▼
S3 충전 완료 팝업
  │ [충전 확인하기] 탭
  ▼
내 하트 페이지 (충전 내역 확인)
```

### 4-2. 정상 플로우 — 스킬 교환권

```
쿠폰 화면
  │ 코드 입력 → [등록] 탭
  │ → 로딩 스피너
  │ → 조회 API (서버가 카카오 상품권으로 판별)
  ▼
S2-B 스킬 확인 팝업 (오버레이)
  │ [사용하기] 탭
  │ → 버튼 로딩 (중복 탭 방지)
  │ → L1 사용 + 쿠폰 발급 API
  ▼
S4 팝업 닫힘 → 토스트 "스킬 이용권이 등록되었어요"
  │ 쿠폰 리스트 최상단에 스킬 이용권 카드 추가
  │ [이용권 카드] 탭
  ▼
스킬 상세 페이지 (♥ 0, 쿠폰 적용)
  │ [지금 보러가기] 탭
  ▼
챗봇 채팅방 (스킬 자동 시작, 무료 이용)
```

### 4-3. 에러 플로우

```
쿠폰 화면 → [등록] → 조회 실패
  ▼
S5 에러 팝업 → [확인] → 쿠폰 화면 (입력 필드 유지)

S2 확인 팝업 → [사용하기] → L1 실패 (또는 내부 실패 → L2 롤백)
  ▼
S5 에러 팝업 → [확인] → 쿠폰 화면 (입력 필드 유지)
```

## 5. 상태별 UI 처리

### 5-1. 로딩 상태

| 시점 | 표현 방식 |
|------|----------|
| 등록 버튼 탭 후 (조회 중) | 화면 중앙 반투명 오버레이 + 스피너 |
| 사용하기 버튼 탭 후 (L1 사용 중) | **사용하기 버튼 로딩 + 비활성화** (중복 탭 방지 필수) |

### 5-2. 버튼 활성/비활성

| 화면 | 조건 | 상태 |
|------|------|------|
| 쿠폰 화면 등록 | 입력 필드 비어있음 | 비활성화 (회색) |
| 쿠폰 화면 등록 | 입력 필드에 값 있음 | 활성화 (분홍색) |
| S2 사용하기 | API 호출 중 | 비활성화 (로딩) |
| S2 취소 | API 호출 중 | 비활성화 |

## 6. API 연동 요약

| 화면 전환 | API | 요청 | 응답 (주요) |
|----------|-----|------|------------|
| 쿠폰 화면 → S2 | `POST /api/coupc-marketing/check` | `{ couponCode }` | `{ valid, productType, productName, heartQuantity?, fixedMenuSeq?, chatbotSeq?, skillName?, couponName, expiryDate }` |
| S2-A → S3 | `POST /api/coupc-marketing/use` | `{ couponCode }` | `{ success, productType:"heart", heartQuantity, productName }` |
| S2-B → S4 | `POST /api/coupc-marketing/use` | `{ couponCode }` | `{ success, productType:"skill", issuedCouponSeq, fixedMenuSeq, chatbotSeq, skillName, productName }` |

> **인증**: 두 API 모두 `@Authorized()` (로그인 필수). 기존 쿠폰 화면이 로그인 상태에서만 접근 가능하므로.

## 7. 디자인 요청 사항

### 신규 에셋

| 에셋 | 용도 | 비고 |
|------|------|------|
| 스킬 이용권 카드 스타일 | 쿠폰 리스트 내 이용권 카드 | 보라색 테두리 + 연보라 배경으로 일반 쿠폰과 구분 |
| 토스트 알림 스타일 | S4 등록 완료 알림 | 검정 배경, 흰 텍스트, 상단 중앙, 2.5초 자동 사라짐 |

### 기존 화면 활용

- **쿠폰 화면**: 기존 그대로 (입력란 + 쿠폰 목록)
- **스킬 상세 페이지**: 기존 그대로 + 쿠폰 적용 시 "♥ 0" + "쿠폰 적용" 배지 표시
- **챗봇 채팅방**: 기존 그대로 + "쿠폰 적용 · 무료로 이용 중" 인디케이터
- **내 하트 페이지**: 기존 그대로 + 충전 내역에 "카카오 상품권 하트 충전" 항목

### 디자인 시 고려 사항

1. **네이티브 UI**: 웹뷰가 아닌 네이티브 화면이므로 기존 앱 디자인 시스템 준수
2. **팝업 디자인**: S2/S3/S5 모두 기존 앱 팝업 스타일과 통일
3. **중복 탭 방지**: 사용하기 버튼은 탭 후 즉시 로딩 상태로 전환 (과금 관련 필수)
4. **스킬 이용권 시각적 구분**: 일반 할인 쿠폰(분홍)과 스킬 이용권(보라)의 배지 색상 차이

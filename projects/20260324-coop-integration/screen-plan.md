> 와이어프레임 v3 확정본 기반. 화면별 목적, 구성 요소, 분기 조건을 정의합니다.

## 2. 진입점 — ✅ 확정

**기존 쿠폰 탭 재사용** + 카카오 딥링크 지원

| 경로 | 설명 |
|------|------|
| 프로필 → 쿠폰 탭 | 기존 쿠폰 입력란을 그대로 사용 |
| 카카오 선물하기 딥링크 | 카카오에서 직접 쿠폰 화면으로 랜딩 |

- 별도 화면 신규 개발 없음
- **서버가 쿠폰 코드 종류를 판별** (`coupon_prefix_rule` 테이블 기반). 클라이언트는 프리픽스 분기 없이 `POST /api/coupon/register` 단일 호출.

## 3. 화면 상세

### 쿠폰 화면 (기존 화면 재사용)

**목적**: 사용자가 쿠폰 코드를 입력하고 보유 쿠폰 목록을 확인하는 화면

**화면 유형**: 네이티브 (iOS / Android / Web)

**구성 요소**:

| 요소 | 설명 |
|------|------|
| 헤더 | "쿠폰 {N}" (보유 쿠폰 수) |
| 입력란 | 플레이스홀더: "쿠폰 코드를 입력해주세요" |
| 등록 버튼 | 입력 시 활성화 (노란색, #FFE967), 비어있으면 비활성화 (회색, #C6C8CC) |
| 힌트 텍스트 | "헬로우봇 쿠폰, 카카오 상품권 등 쿠폰 번호를 입력해보세요." |
| 쿠폰 리스트 | 보유 쿠폰 카드 목록 (기존 + 스킬 이용권) |

**인터랙션**:
- 입력 필드 비어있으면 등록 버튼 비활성화
- 등록 탭 → 로딩 스피너 → `POST /api/coupon/register` 호출 → 응답 `issuedType`별 분기 또는 에러 토스트

**구성 요소** (design-spec.md §S3 확정본 기준):

|------|------|

| 확인 버튼 | **프로필 탭으로 이동** (단일 버튼) |

**인터랙션**:
- 확인 탭 → 쿠폰 화면 닫기 → **프로필 탭**으로 이동 (하트 확인)

### S4. 스킬 이용권 등록 완료

**목적**: 스킬 이용권(100% 할인 쿠폰) 발급 완료를 알리고, 쿠폰 리스트에서 바로 확인할 수 있게 한다

**표시 조건**: register API 응답 `issuedType === "skill"` 성공

> **팝업이 아닌 토스트 + 리스트 업데이트 방식**

**처리 순서**:
1. 쿠폰 입력란 초기화
2. 쿠폰 리스트 **최상단에 스킬 이용권 카드 추가**
3. 상단 **토스트 알림**: "스킬 이용권이 등록되었어요" (2.5초 후 자동 사라짐)
4. 헤더 쿠폰 수 업데이트 ("쿠폰 2" → "쿠폰 3")

**스킬 이용권 카드 구성**:

| 요소 | 설명 |
|------|------|
| 라벨 | "이용권" 태그 (회색, 흰 배경) |
| 할인율 | "100%" (분홍 강조) |
| 쿠폰명 | 스킬명 |
| 챗봇명 | 미노출 |
| 유효기간 | "YYYY.MM.DD HH:mm까지" — **조건부 표시**: 서버 응답 `isUnlimited === true`이면 미표시 (스킬 이용권은 무제한, ISS-021/022) |
| 만료 임박 | "N일 남음" (빨간색, 구분선 포함) — **조건부 표시**: `isUnlimited === true`이면 미표시 |
| 링크 | "스킬 보러가기 >" (보라색) — **하단 우측 정렬 필수** (ISS-023) |
| 카드 스타일 | 흰 배경, 회색 테두리 (기존 쿠폰 카드 형식). 무제한 쿠폰의 경우 유효기간 행 자체가 없으므로 점선 구분선도 숨김 가능 |

**인터랙션**:
- 카드 탭 또는 "스킬 보러가기" 탭 → **스킬 상세 페이지**로 이동 (쿠폰 적용 상태)

### 스킬 상세 페이지 (기존 화면 + 쿠폰 적용)

**목적**: 스킬 이용권이 적용된 상태로 스킬 정보를 확인하고 이용을 시작할 수 있게 한다

- 기존 스킬 상세 페이지 UI 그대로 활용
- **가격 표시 변경**: 원래 가격 취소선 + **♥ 0** 강조 + "쿠폰 적용" 배지
- 하단 CTA: **"지금 보러가기"** (기존 구매 버튼 → 쿠폰 적용으로 무료)

**인터랙션**:
- 지금 보러가기 탭 → **챗봇 채팅방**으로 이동, 쿠폰 자동 적용되어 무료 이용
- 뒤로가기 → 쿠폰 화면으로 복귀

---

### 챗봇 채팅방 (기존 화면)

**목적**: 스킬 이용권 사용으로 스킬이 자동 시작된 채팅방

- 기존 챗봇 채팅방 UI 그대로 활용
- 해당 스킬이 **자동 시작**됨
- "쿠폰 적용 · 무료로 이용 중" 인디케이터 표시 (중앙 배지)
- 별도 하트 차감 없이 진행

### S5. 에러 토스트

**목적**: 쿠폰 등록 실패 시 원인을 안내한다

**표현 방식**: 하단 중앙 토스트, 2.5초 후 자동 사라짐

**에러 메시지**: `error.message` 필드를 그대로 표시 (서버에서 i18n 처리)

| 에러코드 | 사용자 메시지 |
|---------|---------------|
| CM_001 | 유효하지 않은 쿠폰이에요 |
| CM_002 | 기간이 만료된 쿠폰이에요 |
| CM_003 | 이미 사용된 쿠폰이에요 |
| CM_004 | 상품을 찾을 수 없어요 |
| CM_005 | 일시적인 서비스 오류가 발생했어요 |
| CM_006 | 일시적인 통신 오류가 발생했어요 |
| CM_007 | 하트 충전에 실패했어요 (쿠폰 원복됨, 재시도 가능) |
| CM_008 | 스킬 이용권 발급에 실패했어요 (쿠폰 원복됨, 재시도 가능) |
| CM_010 | 결제가 취소된 쿠폰이에요 |
| CO_APP_UPDATE_REQUIRED | 앱 업데이트가 필요한 쿠폰이에요. (Phase 1 현재: 구버전 앱에서만 발생. 미래 확장 시 신버전 앱도 미지원 `couponType` 수신 시 동일 메시지 표시 — architecture.md §7 참조) |

**인터랙션**:
- 토스트 자동 사라짐 후 쿠폰 화면 유지 (입력 필드 유지, 재시도 가능)

## 4. 화면 전환 상세

### 4-1. 정상 플로우 — 하트 충전권

```
쿠폰 화면
  │ 코드 입력 → [등록] 탭
  │ → 로딩 스피너
  │ → POST /api/coupon/register (서버가 종류 판별 + L0+L1 원샷 처리)
  │ → 응답: issuedType = "heart"
  ▼
S3 충전 완료 팝업
  │ [확인] 탭
  ▼
프로필 탭 (하트 확인)
```

### 4-2. 정상 플로우 — 스킬 교환권

```
쿠폰 화면
  │ 코드 입력 → [등록] 탭
  │ → 로딩 스피너
  │ → POST /api/coupon/register (서버가 종류 판별 + L0+L1 원샷 처리)
  │ → 응답: issuedType = "skill"
  ▼
S4 토스트 "스킬 이용권이 등록되었어요"
  │ 쿠폰 리스트 최상단에 스킬 이용권 카드 추가
  │ [이용권 카드] 탭
  ▼
스킬 상세 페이지 (♥ 0, 쿠폰 적용)
  │ [지금 보러가기] 탭
  ▼
챗봇 채팅방 (스킬 자동 시작, 무료 이용)
```

### 4-3. 에러 플로우

```
쿠폰 화면 → [등록] → POST /api/coupon/register 실패 (HTTP 4xx/5xx)
  ▼
S5 에러 토스트 (2.5초, error.message 표시) → 쿠폰 화면 유지 (입력 필드 유지)
```

### 4-4. 구버전 앱 플로우 (참고)

```
구버전 앱 → [등록] (90/91 코드 입력)
  │ → POST /api/coupon (구버전은 신규 API 모름)
  │ → 서버 가드: HTTP 406 + CO_APP_UPDATE_REQUIRED
  ▼
기존 에러 토스트: "앱 업데이트가 필요한 쿠폰이에요." → 쿠폰 화면 유지
```

## 6. API 연동 요약

| 화면 전환 | API | 요청 | 응답 (주요) |
|----------|-----|------|------------|
| 쿠폰 화면 → 분기 UI | `POST /api/coupon/register` | `{ code }` | 성공: `{ resultType: "ISSUED", issuedType: "coupon"\|"heart"\|"skill", ... }` / 실패: `{ error: { code, message } }` |

> **인증**: API는 `@Authorized()` (로그인 필수). 기존 쿠폰 화면이 로그인 상태에서만 접근 가능하므로.

## 7. 디자인 요청 사항

### 신규 에셋

| 에셋 | 용도 | 비고 |
|------|------|------|
| 스킬 이용권 카드 스타일 | 쿠폰 리스트 내 이용권 카드 | 기존 쿠폰 카드 형식 + "이용권" 라벨 + 100% 할인율 + "스킬 보러가기" 링크 |
| 토스트 알림 스타일 | S4 등록 완료 알림 | 검정 배경, 흰 텍스트, 하단 중앙, 2.5초 자동 사라짐 |

### 기존 화면 활용

- **쿠폰 화면**: 기존 그대로 (입력란 + 쿠폰 목록)
- **스킬 상세 페이지**: 기존 그대로 + 쿠폰 적용 시 "♥ 0" + "쿠폰 적용" 배지 표시
- **챗봇 채팅방**: 기존 그대로 + "쿠폰 적용 · 무료로 이용 중" 인디케이터
- **내 하트 페이지**: 기존 그대로 + 충전 내역에 "카카오 상품권 하트 충전" 항목

### 디자인 시 고려 사항

1. **네이티브 UI**: 웹뷰가 아닌 네이티브 화면이므로 기존 앱 디자인 시스템 준수
2. **팝업 디자인**: S3 완료 팝업은 기존 앱 팝업 스타일과 통일, S5는 토스트. S2 확인 팝업은 **1단계 원샷 전환으로 제거됨** (2026-04-19) — 디자인/구현 대상 아님.
3. **중복 탭 방지**: 등록 버튼은 탭 후 즉시 로딩 상태로 전환 (과금/중복 발급 방지 필수)
4. **스킬 이용권 카드**: 기존 쿠폰 카드 형식 + "이용권" 라벨 + 100% 할인율 + "스킬 보러가기" 링크

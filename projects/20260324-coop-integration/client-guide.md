# 클라이언트 개발 가이드 — 카카오 선물하기 상품권

## 개요

카카오 선물하기로 받은 상품권(쿠폰번호)을 기존 쿠폰 탭에서 입력하여 **하트 충전** 또는 **스킬 이용권 발급**을 받는 기능입니다. 별도 화면 신규 개발 없이 기존 쿠폰 화면과 스킬 상세 페이지를 재사용합니다.

> **화면 기획서**: [screen-plan.md](./screen-plan.md)
> **API 명세**: [api-spec.md](./api-spec.md)
> **와이어프레임**: [wireframe-v3.html](./wireframe-v3.html)

---

## 연동 순서

### 1단계: 쿠폰 조회 (check)

사용자가 쿠폰번호를 입력하고 등록 버튼을 탭하면 조회 API를 호출합니다.

```
POST /api/coop/check
Authorization: Bearer {token}
Content-Type: application/json

{ "couponCode": "901914216415" }
```

**응답 분기**:

| 조건 | 처리 |
|------|------|
| `valid: true` + `productType: "heart"` | S2-A 하트 충전권 확인 팝업 표시 |
| `valid: true` + `productType: "skill"` | S2-B 스킬 교환권 확인 팝업 표시 |
| `valid: false` | S5 에러 팝업 표시 (`errorMessage` 사용) |

**check 응답 예시 — 하트 충전권**:
```json
{
  "data": {
    "valid": true,
    "productType": "heart",
    "productName": "카카오 하트 충전권 5천원",
    "heartQuantity": 25,
    "couponName": "카카오 선물하기 하트 충전권"
  }
}
```

**check 응답 예시 — 스킬 교환권**:
```json
{
  "data": {
    "valid": true,
    "productType": "skill",
    "productName": "그 사람과 나의 사주 궁합",
    "fixedMenuSeq": 2166,
    "chatbotSeq": 123,
    "skillName": "그 사람과 나의 사주 궁합",
    "couponName": "카카오 선물하기 스킬 교환권"
  }
}
```

**check 응답 예시 — 실패**:
```json
{
  "data": {
    "valid": false,
    "errorCode": "CM_001",
    "errorMessage": "유효하지 않은 쿠폰입니다"
  }
}
```

### 2단계: 쿠폰 사용 (use)

확인 팝업에서 "사용하기" 버튼을 탭하면 사용 API를 호출합니다.

```
POST /api/coop/use
Authorization: Bearer {token}
Content-Type: application/json

{ "couponCode": "901914216415" }
```

**use 응답 예시 — 하트 충전 성공**:
```json
{
  "data": {
    "success": true,
    "productType": "heart",
    "heartQuantity": 25,
    "productName": "카카오 하트 충전권 5천원"
  }
}
```

**use 응답 예시 — 스킬 이용권 발급 성공**:
```json
{
  "data": {
    "success": true,
    "productType": "skill",
    "issuedCouponSeq": 456,
    "fixedMenuSeq": 2166,
    "chatbotSeq": 123,
    "skillName": "그 사람과 나의 사주 궁합",
    "productName": "그 사람과 나의 사주 궁합",
    "message": "스킬 이용권이 등록되었어요"
  }
}
```

### 3단계: 결과 처리

| productType | 성공 시 처리 |
|-------------|------------|
| `heart` | S3 하트 충전 완료 팝업 → "충전 확인하기" → 내 하트 페이지 이동 |
| `skill` | S2-B 팝업 닫기 → 토스트 "스킬 이용권이 등록되었어요" → 쿠폰 리스트에 이용권 카드 추가 |

## 화면별 구현 상세

### 쿠폰 화면 (기존 화면 수정)

- 기존 쿠폰 입력란 그대로 사용
- 힌트 텍스트 변경: **"헬로우봇 쿠폰, 카카오 상품권 등 쿠폰 번호를 입력해보세요."**
- 등록 버튼 탭 시 서버가 쿠폰 형식을 판별 (클라이언트에서 분기 불필요)
- check API 응답의 `valid` 값으로 성공/실패 분기

### S2-A. 하트 충전권 확인 팝업

| 요소 | 데이터 소스 |
|------|-----------|
| 하트 아이콘 | 공통 에셋 (♥ 분홍 원형) |
| 상품명 | check 응답의 `productName` |
| 충전 수량 | check 응답의 `heartQuantity` → "♥ {N}개" |
| 안내 문구 | "이 쿠폰을 사용하면 하트 {N}개가 충전됩니다. 사용하시겠습니까?" |
| 취소 버튼 | 팝업 닫기 |
| 사용하기 버튼 | use API 호출 |

### S2-B. 스킬 교환권 확인 팝업

| 요소 | 데이터 소스 |
|------|-----------|
| 배너 이미지 | `chatbotSeq`로 챗봇 배너 이미지 조회 |
| 상품명 | check 응답의 `productName` |
| 챗봇명 | `chatbotSeq`로 챗봇명 조회 |
| 안내 문구 | "이 쿠폰을 스킬 이용권으로 교환합니다. 사용하시겠습니까?" |
| 취소 버튼 | 팝업 닫기 |
| 사용하기 버튼 | use API 호출 |

### S3. 하트 충전 완료 팝업

| 요소 | 데이터 소스 |
|------|-----------|
| 완료 아이콘 | ✓ (초록 원형) |
| 메시지 | use 응답의 `heartQuantity` → "하트 {N}개가 충전되었어요" |
| 충전 확인하기 버튼 | 내 하트 페이지로 이동 |

### S4. 스킬 이용권 등록 완료

use API 성공 후 순서:
1. S2-B 팝업 닫기
2. 쿠폰 입력란 초기화
3. 쿠폰 리스트 최상단에 **스킬 이용권 카드** 추가
4. 토스트: "스킬 이용권이 등록되었어요" (2.5초 후 자동 사라짐)
5. 헤더 쿠폰 수 +1 업데이트

**스킬 이용권 카드**:
- 배지: "스킬 이용권" (보라색)
- 이름: use 응답의 `productName` + " 이용권"
- 설명: 챗봇명
- 카드 탭 → 스킬 상세 페이지 이동 (`fixedMenuSeq` 사용)

### S5. 에러 팝업

| 요소 | 데이터 소스 |
|------|-----------|
| 에러 아이콘 | ! (빨간 원형) |
| 메시지 | check 응답의 `errorMessage` 또는 use 에러의 `message` |
| 확인 버튼 | 팝업 닫기 → 쿠폰 화면 복귀 (입력 필드 유지) |

### S6. 로그인 안내 팝업

미로그인(anonymous) 사용자가 쿠폰 입력란을 포커스할 때 표시합니다.

| 요소 | 설명 |
|------|------|
| 경고 아이콘 | ! (빨간 원형, 24px) |
| 제목 | "로그인하지 않고 쿠폰을 등록하면 위험해요" |
| 설명 | "소중한 선물이 사라질 수 있어요 꼭 로그인하고 입력해 주세요" |
| 로그인하기 버튼 | 로그인 페이지 이동 (fallbackUrl=/coupon) |
| 다음에 할래요 | 팝업 닫기 |

**동작 규칙**:
- 오버레이(dim) 클릭으로는 닫히지 않음 — 반드시 버튼으로만 닫기
- 세션 내 1회만 노출: "다음에 할래요" 클릭 시 세션 동안 재노출하지 않음
- 로그인 후 재진입하면 anonymous가 아니므로 팝업 미표시

## 에러 처리

### check API 에러

check API는 **항상 HTTP 200**을 반환합니다. `valid: false`일 때 `errorMessage`를 팝업에 표시합니다.

| errorCode | 사용자 메시지 |
|-----------|-------------|
| CM_001 | 유효하지 않은 쿠폰입니다 |
| CM_002 | 기간이 만료된 쿠폰입니다 |
| CM_003 | 이미 사용된 쿠폰입니다 |
| CM_004 | 상품을 찾을 수 없습니다 |
| CM_006 | 일시적인 통신 오류가 발생했습니다 |

### use API 에러

use API는 실패 시 **HTTP 4xx/5xx**를 반환합니다. 에러 응답의 `message` 필드를 팝업에 표시합니다.

```json
{
  "code": "CM_007",
  "message": "하트 충전에 실패했습니다",
  "reason": "하트 충전에 실패했습니다"
}
```

| HTTP | errorCode | 사용자 메시지 |
|------|-----------|-------------|
| 400 | CM_001 | 유효하지 않은 쿠폰입니다 |
| 400 | CM_002 | 기간이 만료된 쿠폰입니다 |
| 400 | CM_003 | 이미 사용된 쿠폰입니다 |
| 400 | CM_005 | 쿠프마케팅 API 오류가 발생했습니다 |
| 404 | CM_004 | 상품을 찾을 수 없습니다 |
| 500 | CM_006 | 일시적인 통신 오류가 발생했습니다 |
| 500 | CM_007 | 하트 충전에 실패했습니다 |
| 500 | CM_008 | 스킬 이용권 발급에 실패했습니다 |
| 500 | CM_009 | 쿠폰 스펙을 찾을 수 없습니다 |

> CM_007, CM_008 발생 시 서버가 자동으로 쿠프마케팅 승인 취소(L2)를 시도합니다. 쿠폰이 원복되므로 사용자에게 "다시 시도해주세요"를 안내해도 됩니다.

## 주의사항

### 중복 탭 방지 (필수)

- **사용하기 버튼**: 탭 즉시 로딩 상태 + 비활성화
- use API는 쿠프마케팅 외부 API를 호출하므로 응답이 느릴 수 있음 (최대 15초)
- 중복 호출 시 과금 문제 발생 가능 → 반드시 클라이언트에서 차단

### 쿠폰번호 형식

- 클라이언트에서 쿠폰 형식 판별 불필요 (서버가 판별)
- 기존 쿠폰과 카카오 상품권 모두 동일한 입력란 사용
- 붙여넣기 지원 필수 (카카오 선물하기에서 복사해오는 동선)

### 인증

- 두 API 모두 `@Authorized()` (로그인 필수)
- 기존 쿠폰 화면이 로그인 상태에서만 접근 가능하므로 추가 처리 불필요

### 딥링크

- 카카오 선물하기에서 쿠폰 화면으로 직접 랜딩하는 딥링크 지원 필요
- 딥링크 URL 스킴은 추후 확정

## 플로우 요약

```
[쿠폰 화면] ─ 코드 입력 ─ [등록]
     │
     ▼
POST /api/coop/check
     │
     ├── valid: false ──→ [S5 에러 팝업] → [확인] → 쿠폰 화면
     │
     ├── heart ──→ [S2-A 확인 팝업] ─ [사용하기]
     │                                    │
     │                              POST /api/coop/use
     │                                    │
     │                     ├── 성공 ──→ [S3 완료 팝업] → 내 하트 페이지
     │                     └── 실패 ──→ [S5 에러 팝업]
     │
     └── skill ──→ [S2-B 확인 팝업] ─ [사용하기]
                                         │
                                   POST /api/coop/use
                                         │
                          ├── 성공 ──→ 토스트 + 이용권 카드 추가
                          │              │ 카드 탭
                          │              ▼
                          │         스킬 상세 (♥0) → 챗봇 채팅방
                          └── 실패 ──→ [S5 에러 팝업]
```

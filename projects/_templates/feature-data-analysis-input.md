# 신기능 데이터 분석 — 입력 템플릿

> **사용법**: 새 기능의 데이터 분석·이벤트 설계를 시작할 때 본 템플릿의 항목을 채워 `/dev-data` 에 전달.
>
> **출력**: `/dev-data` 가 본 입력 + 데이터 카탈로그(`infra-map.md` / `feature-performance-measurement.md` / `event-design-guide.md` / `event-catalog.md` / `metric-dictionary.md`) 를 참조해 성과 측정 설계 문서(`projects/{프로젝트}/architecture.md` 의 §데이터 섹션) 를 작성.
>
> **TBD/모름 항목 OK** — 모르는 부분은 "TBD" 또는 "확인 필요" 로 두고 진행. `/dev-data` 가 갭으로 식별해 추가 질문을 던집니다.

   - 갭(`issues.md`, `external-tasks.md`) 식별
   - `architecture.md` §데이터 섹션에 측정 지표·이벤트 스펙·마트 변경·분석 쿼리 작성

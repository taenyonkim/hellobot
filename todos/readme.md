# TODO 상세 파일

워크스페이스 루트의 [`TODO.md`](../TODO.md)는 한 줄 인덱스이고, 각 TODO의 상세 컨텍스트·진행 상황은 이 디렉토리에 `TODO-NNN-{slug}.md` 파일로 보관합니다.

## 목적

하나의 TODO는 한 번의 대화 턴/세션 안에서 끝나지 않을 수 있습니다. 다음 세션에서 코디네이터가 **이전 작업을 다시 뒤지지 않고** 곧바로 이어서 진행할 수 있도록, 각 TODO의 컨텍스트와 현재 위치를 별도 파일에 누적합니다.

## 운영 규칙

- 새 TODO 등록 시 **반드시** 이 디렉토리에 상세 파일을 함께 생성합니다 (사소한 일도 동일).
- 파일명: `TODO-NNN-{kebab-case-slug}.md` (예: `TODO-001-onboarding-script-fix.md`).
- TODO.md의 한 줄 항목에서 상세 파일로 링크: `- [ ] TODO-001 제목 — [상세](todos/TODO-001-slug.md)`.
- TODO 완료 후에도 상세 파일은 보존합니다. 완료 항목이 TODO.md에서 잘려나가도 상세 파일은 남깁니다 (참고용).
- 진행이 멈춰 있다가 재개할 때, 코디네이터는 **가장 먼저 `현재 상태`와 `다음 단계` 섹션을 읽고** 그 지점부터 작업을 이어갑니다.

## 표준 템플릿

```markdown
# TODO-NNN 제목

**상태**: 진행 중 | 대기 (사유) | 완료 (YYYY-MM-DD)
**등록**: YYYY-MM-DD
**담당**: 코디네이터 | /dev-server 위임 | …
**관련**: projects/YYYYMMDD-name/ (프로젝트화된 경우)

## 컨텍스트
사용자 요청 원문, 배경, 제약, 관련 리포·문서 링크.
한 번 적으면 거의 바뀌지 않는 영역.

## 현재 상태
어디까지 했고 지금 무엇을 기다리고 있는지 한두 문단.
**재개 시 코디네이터가 가장 먼저 읽는 곳** — 항상 최신으로 유지.

## 다음 단계
- [ ] 재개 시 즉시 해야 할 액션
- [ ] 그 다음
- [ ] …

## 진행 로그
시간순 추가만(append-only). 과거 항목 수정 금지.

- YYYY-MM-DD HH:MM — 시작, 관련 파일 식별
- YYYY-MM-DD HH:MM — A 분석 완료, B 확인 필요
- YYYY-MM-DD HH:MM — 사용자 확인 대기 중
```

## 작성 원칙

- **현재 상태**와 **다음 단계**는 매 세션 종료 전 갱신. 이 두 섹션이 어긋나면 재개 시 혼란.
- **진행 로그**는 추가만. 과거 기록을 다듬거나 지우지 말 것 — 의사결정 흐름 추적용.
- 깊은 코드 탐색·설계 결정이 누적되면 그 자체가 프로젝트화 신호. `projects/`로 승격하고 상세 파일은 프로젝트 링크만 남기는 형태로 축약.
- 사용자에게 무엇을 물어봤는지, 무엇을 답받았는지 진행 로그에 남길 것 — 같은 질문 반복 방지.

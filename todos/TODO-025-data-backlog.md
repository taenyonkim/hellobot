## 컨텍스트

데이터 분석·측정·인프라 관련 **할일·아이디어·요청을 누적**하는 umbrella TODO. 단건으로 즉시 처리하기엔 작거나, 우선순위가 낮거나, 다른 작업과 묶어 처리할 가치가 있는 항목들의 보관소.

### 운영 규칙

- **항목 추가**: 발견·요청 시 본 파일 `## 백로그 항목` 표에 즉시 추가 (ID·제목·간단 설명·등록일)
- **승격 조건** (다음 중 하나에 해당하면 별도 TODO·프로젝트로 분리):
  - 마감일이 생김
  - 즉시 실행 단계로 진입 (`/dev-data` 위임 결정)
  - 다른 항목과 결합해 규모가 커짐 (프로젝트화 시점)
- **승격 시**: 백로그 표에서 행을 유지하되 상태 컬럼을 「→ TODO-NNN 승격」 또는 「→ projects/...」로 갱신
- **드롭**: 무의미해진 항목은 「드롭 (사유)」로 마킹 (행 삭제 금지 — 결정 흐름 보존)

### 백로그 vs 별도 TODO 판단 기준

| 상황 | 처리 |
|------|------|
| 단건 작업, 즉시 실행 가능, 마감 있음 | 별도 TODO 등록 (백로그 거치지 않음) |
| 아이디어 단계, 누구·언제 명확하지 않음 | **백로그** |
| 다른 항목과 묶어 처리하면 효율적 | **백로그** (묶임 시점에 승격) |
| 정기·반복적 점검 | **백로그** (가장 적합) |
| 사용자 결정 필요한데 결정 시점 미정 | **백로그** |
